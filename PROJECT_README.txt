【專案目的】
每天早上自動產出一份包含三大期貨（小道瓊、小那指、台指期）
的專業分析報告，並透過 GitHub Pages 網站呈現。

───────────────────────────────────────────────────────────────
  檔案結構說明
───────────────────────────────────────────────────────────────

futures-report/
│
├── .github/
│   └── workflows/
│       └── fetch_data.yml        ← GitHub Actions 排程設定
│                                    每天台灣時間 05:50（UTC 21:50）
│                                    週一到週五自動執行 fetch_data.py
│
├── fetch_data.py                 ← 爬蟲主腳本
│                                    負責抓取所有數字型數據並計算技術指標
│                                    執行完畢後將結果存成 data.json 推送到 GitHub
│
├── data.json                     ← 爬蟲每天自動產出的數據檔
│                                    內容包含：
│                                    - YM1 / NQ1 / TXF1 點位、漲跌幅、成交量
│                                    - 各期貨技術指標（RSI / MA / MACD / 布林通道 / ATR）
│                                    - 各期貨五日歷史收盤價
│                                    - VIX / 美元指數 DXY / 布蘭特原油
│                                    此檔案由程式自動覆寫，不可手動編輯
│
├── template.json                 ← report.json 的格式範本
│                                    定義 report.json 的所有欄位名稱與層級結構
│                                    Manus 讀取此檔案確保輸出格式正確
│                                    此檔案為固定格式，不可隨意修改欄位名稱
│
├── report.json                   ← Manus 每天產出的分析報告
│                                    內容包含：
│                                    - 市場情緒（VIX / DXY / 油價）
│                                    - 期貨數據（點位 / 支撐壓力 / 技術指標）
│                                    - 昨日走勢分析 / 經濟數據
│                                    - 今日機率評估 / 操作建議
│                                    - 新聞摘要 / 風險提示
│                                    此檔案由 Manus 自動覆寫，不可手動編輯
│
├── index.html                    ← 報告網站前端
│                                    讀取 report.json 並渲染成視覺化報告
│                                    透過 GitHub Pages 對外公開顯示
│
└── CNAME                         ← GitHub Pages 自訂網域設定

───────────────────────────────────────────────────────────────
  每日自動化流程
───────────────────────────────────────────────────────────────

05:50 台灣時間
  └─ GitHub Actions 觸發 fetch_data.py
       │
       ├─ 抓取 YM1（YM=F）點位與歷史  ← Yahoo Finance API
       ├─ 抓取 NQ1（NQ=F）點位與歷史  ← Yahoo Finance API
       ├─ 抓取 TXF1 點位              ← TAIFEX 官網
       ├─ 抓取 TXF1 技術指標歷史      ← Yahoo Finance（^TWII）
       ├─ 抓取 VIX / DXY / 油價       ← Yahoo Finance API
       └─ 計算技術指標（RSI / MA / MACD / 布林通道 / ATR）
            │
            └─ 存成 data.json 推送到 GitHub

06:00 台灣時間
  └─ Manus 排程任務觸發
       │
       ├─ 讀取 data.json（準確數據）
       ├─ 讀取 template.json（輸出格式）
       ├─ 自行搜尋：昨日新聞、經濟數據、市場事件
       └─ 結合數據與新聞進行分析
            │
            └─ 輸出 report.json 推送到 GitHub

隨時
  └─ 使用者開啟報告網站
       └─ index.html 讀取 report.json 渲染顯示

  數據來源說明

數據類型          來源                    說明

                  query1.finance.yahoo    closes[-3] 為前一收盤
                  .com/v8/finance/chart   夜盤即時價不採用

                  www.taifex.com.tw       抓 TX 當日收盤價
─────────────────────────────────────────────────────────────

                                          用於計算 RSI / MA 等
─────────────────────────────────────────────────────────────
VIX               Yahoo Finance（^VIX）   regularMarketPrice

───────────────────────────────────────────────────────────────
  技術指標計算說明
───────────────────────────────────────────────────────────────

指標            計算方式                    用途
─────────────────────────────────────────────────────────────
RSI14           14日相對強弱指標            超買超賣判斷
MA5             5日收盤均價                 短期趨勢
MA20            20日收盤均價               中期趨勢
MA60            60日收盤均價               長期趨勢
MACD            12/26/9 指數移動平均        趨勢轉折訊號
布林通道         20日均線 ± 2倍標準差       波動區間
ATR14           14日平均真實波幅            波動度 / 停損參考
Volume MA5      5日成交量均量               量價配合判斷
─────────────────────────────────────────────────────────────
所有指標以最近一次收盤為基準計算（不含當天夜盤）

───────────────────────────────────────────────────────────────
  注意事項
───────────────────────────────────────────────────────────────

1. 五日歷史收盤價
   YM1 / NQ1 的五日歷史來自 Yahoo Finance 近月合約，
   與 TradingView 連續合約（YM1! / NQ1!）可能有價差，
   原因是期貨換倉（Roll Over）時合約不同。
   此數據僅供趨勢方向參考，不用於精確比對。

2. GitHub Actions 免費額度
   每月 2000 分鐘，每天跑一次約 30 秒，
   全年使用量約 130 分鐘，遠低於免費額度上限。

3. Manus 點數消耗
   爬蟲已提供所有數字型數據與技術指標，
   Manus 只需負責新聞搜尋與分析文字，
   點數消耗比原本大幅減少。

4. 週末不執行
   GitHub Actions 排程設定為週一到週五（1-5），
   週六週日期貨休市，不需要跑。
